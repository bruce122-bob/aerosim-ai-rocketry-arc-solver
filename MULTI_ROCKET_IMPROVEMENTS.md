# 多火箭数据分析系统改进

基于PDF报告《非Subscale火箭发射数据分析报告（更新版）》的改进方案

## 📊 改进概述

根据报告中6款火箭的性能数据分析，我们实现了以下功能：

### 1. 多火箭设计对比分析 ✅

**功能位置**: `Analysis` 标签页 → `多火箭对比` 子标签

**功能说明**:
- 同时对比分析6款火箭的性能指标
- 自动计算平均高度、标准差、稳定性评分
- 可视化展示各火箭的性能对比

**支持的火箭**:
- Cheese Rocket (5次发射)
- P52 Rocket (10次发射)
- Project Epsilon (4次发射，最高853ft)
- New Design (13次发射，最稳定)
- Vincent Rocket (1次发射)
- White (YK 4.0) (2次发射)

### 2. 发动机序列号跟踪 ✅

**实现位置**: `services/multiRocketAnalysis.ts`

**功能说明**:
- 跟踪发动机序列号（如F42-6T序列号124）
- 分析特定序列号发动机的故障模式
- 识别发动机与硬件故障的关联

**关键发现**:
- F42-6T（序列号124）与舵面损坏、降落伞故障高度相关
- 使用该发动机的火箭故障率达40%

### 3. 硬件故障预测系统 ✅

**实现位置**: `predictHardwareFailureRisk()` 函数

**功能说明**:
- 基于发动机型号和序列号预测故障风险
- 识别高风险组件（舵面、降落伞、结构）
- 提供针对性的改进建议

**预测规则**:
- F42-6T（序列号124）: 高风险（40%故障率）
- 高推重比（>15）: 中等风险（结构应力）
- 自动检测并警告潜在问题

### 4. 设计稳定性分析 ✅

**实现位置**: `analyzeRocketPerformance()` 函数

**功能说明**:
- 基于标准差计算稳定性评分
- 识别性能波动较大的设计
- 对比不同设计的可靠性

**稳定性指标**:
- 稳定性评分 = 100 - (标准差/平均高度) × 100
- 评分越高，性能越稳定
- New Design: 最稳定（标准差30.8ft）
- Project Epsilon: 性能最高但稳定性较差（标准差58.7ft）

### 5. 质量-性能关系分析 ✅

**实现位置**: `calculateMassPerformanceCorrelation()` 函数

**功能说明**:
- 计算质量与性能的相关系数
- 可视化质量-高度散点图
- 识别轻量化设计的优势

**关键发现**:
- 相关系数: -0.3 到 -0.5（负相关）
- 轻量化设计有助于提高性能
- Project Epsilon（579.5g）达到最高853ft
- New Design（569.5g）性能稳定

### 6. 批量数据导入功能 🔄

**实现位置**: `createRocketDesignFromReport()` 函数

**功能说明**:
- 支持从PDF报告数据创建火箭设计记录
- 可扩展支持CSV/JSON批量导入
- 自动解析发射记录和故障信息

**数据结构**:
```typescript
interface RocketDesignRecord {
    name: string;
    motor?: string;
    motorSerial?: string;
    launches: LaunchRecord[];
}
```

## 📈 可视化功能

### 图表类型

1. **平均高度对比柱状图**
   - 显示各火箭的平均、最小、最大高度
   - 快速识别最佳性能设计

2. **稳定性分析图**
   - 稳定性评分 vs 标准差
   - 识别需要改进的设计

3. **质量-性能散点图**
   - 质量（g）vs 高度（ft）

   - 颜色编码（绿/黄/红）

2. 点击 `Analysis` 标签
3. 选择 `多火箭对比` 子标签
4. 查看对比分析结果

- 系统自动分析当前火箭设计
- 显示故障风险等级（低/中/高）
- 提供针对性改进建议
