# 模拟确定性说明

## 问题诊断

如果相同条件下模拟结果不一致，可能的原因：

### 1. 环境参数变化
**最常见原因**：环境参数（风速、温度、压力等）在两次运行之间被修改了。

**检查方法**：
- 查看浏览器控制台的 `[SIMULATION]` 日志
- 对比两次运行的输入参数是否完全相同
- 特别注意：风速、温度、压力、湿度

**从你的日志看**：
- 第一次运行：Wind: 6.8m/s @ 22.5°
- 第二次运行：Wind: 4.3m/s @ 22.5°

**结论**：风速不同（6.8 vs 4.3），所以结果不同是正常的。

### 2. 随机性来源
模拟默认是**完全确定性的**，除非启用了随机阵风。

**检查随机性**：
- `enableRandomGusts` 默认是 `false`
- 如果看到警告 `⚠️ Random gusts enabled`，说明启用了随机性
- 只有在运行蒙特卡洛模拟时才会启用随机性

### 3. 浮点数精度
理论上，浮点数运算可能有微小的精度差异，但在实际应用中：
- RK4积分方法非常稳定
- 相同输入应该产生相同输出（误差 < 0.01%）

## 确保确定性

### 方法1：检查输入参数
每次运行模拟前，查看控制台日志：
```
[SIMULATION] Starting simulation with parameters:
[SIMULATION]   Wind Speed: X.XX m/s
[SIMULATION]   Temperature: XX.X °C
[SIMULATION]   Pressure: XXXX.X hPa
...
```

确保所有参数都相同。

### 方法2：使用固定种子（如果需要随机性）
如果运行蒙特卡洛模拟，可以使用固定种子确保可重复性：
```typescript
// 在 Monte Carlo 函数中使用固定种子
const seed = 12345; // 固定种子
// 使用种子生成随机数
```

### 方法3：验证确定性
运行两次完全相同的模拟，结果应该：
- Apogee 完全相同（误差 < 0.1 ft）
- 所有数据点完全相同

## 常见问题

### Q: 为什么风速会变化？
A: 可能的原因：
1. UI中的风速滑块被移动了
2. 环境参数被其他代码修改了
3. 使用了自动环境检测（如果有的话）

**解决方法**：手动设置固定的环境参数，不要使用自动检测。

### Q: 如何确保完全相同的输入？
A: 
1. 在运行模拟前，记录所有输入参数
2. 使用控制台日志对比两次运行的参数
3. 确保火箭配置、环境参数、发射角度都相同

### Q: 模拟结果有微小差异（< 0.1%）正常吗？
A: 
- 理论上不应该有差异
- 如果差异 < 0.01%，可能是浮点数精度问题（可忽略）
- 如果差异 > 0.1%，说明输入参数不同

模拟代码保证：
- ✅ 默认情况下完全确定性（`enableRandomGusts = false`）

- ✅ RK4积分方法稳定且可重复

唯一可能的随机性来源：
- `enableRandomGusts = true`（仅用于蒙特卡洛模拟）
